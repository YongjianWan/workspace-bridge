# ADR-001: 认证方案选择

## 状态

已接受

## 背景

我们需要为用户系统设计一个认证机制。考虑到系统的分布式特性和可扩展性需求，需要选择一个合适的认证方案。

## 决策

我们决定使用JWT（JSON Web Token）进行无状态认证。

### 理由

1. **无状态**: 不需要在服务端存储session，便于水平扩展
2. **跨域支持**: JWT天然支持跨域认证
3. **标准化**: JWT是RFC 7519标准，有丰富的库支持
4. **自包含**: Token中包含用户必要信息，减少数据库查询

### 替代方案

- **Session-based**: 需要服务端存储，不利于扩展
- **OAuth 2.0**: 过于复杂，适合第三方登录场景
- **API Key**: 不适合有状态的用户认证

## 后果

### 正面

- 系统易于水平扩展
- 认证服务可以独立部署
- 支持移动端和Web端统一认证

### 负面

- Token无法主动失效（需要额外机制如黑名单）
- Token体积比session id大
- 需要处理Token刷新机制
