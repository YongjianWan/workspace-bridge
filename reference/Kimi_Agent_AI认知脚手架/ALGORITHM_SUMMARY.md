## 核心算法列表

### 1. 符号索引算法 (Symbol Indexing)

**功能**: 基于Tree-sitter构建跨语言符号索引

**数据结构**:

- `Symbol`: 符号定义（名称、类型、位置、签名等）
- `SymbolIndex`: 多索引哈希表（名称、文件、类型、限定名）

**复杂度**:

| 操作         | 时间    | 空间 |
| ------------ | ------- | ---- |
| 添加符号     | O(1)    | O(1) |
| 按名称查询   | O(1)    | O(k) |
| 按文件查询   | O(1)    | O(k) |
| 全文索引构建 | O(F×n) | O(S) |

**多语言支持**: Python, TypeScript, Go

---

### 2. 依赖图构建算法 (Dependency Graph)

**功能**: 从import/export关系构建有向图

**数据结构**:

- `DependencyGraph`: 有向图（邻接表 + 反向邻接表）
- `DependencyEdge`: 依赖边（源、目标、类型、元数据）

**复杂度**:

| 操作       | 时间      | 空间   |
| ---------- | --------- | ------ |
| 添加节点   | O(1)      | O(1)   |
| 添加边     | O(1)      | O(1)   |
| 获取依赖   | O(1)      | O(d)   |
| 构建完整图 | O(F×n+E) | O(V+E) |

**多语言支持**: Python, TypeScript, Go

---

### 3. PageRank中心性计算

**功能**: 识别代码库中的核心节点

**算法**: 迭代PageRank计算

**复杂度**:

| 操作         | 时间       | 空间   |
| ------------ | ---------- | ------ |
| 构建转移矩阵 | O(E)       | O(V²) |
| PageRank迭代 | O(k×V²)  | O(V²) |
| 识别核心节点 | O(V log V) | O(V)   |

**优化**: 使用稀疏矩阵可将空间复杂度降至O(E)

---

### 4. 影响传播算法 (Impact Propagation)

**功能**: 计算修改的二级传递效应

**算法**: BFS遍历 + 深度衰减

**复杂度**:

| 操作       | 时间          | 空间    |
| ---------- | ------------- | ------- |
| 单节点分析 | O(V+E)        | O(V)    |
| 多节点分析 | O(k×(V+E))   | O(k×V) |
| 生成报告   | O(k×V log V) | O(k×V) |

**影响级别**:

- 直接影响: 深度=0
- 间接影响: 深度=1
- 传递影响: 深度≥2

---

### 5. 死代码检测集成

**功能**: 集成vulture/ts-prune/unused进行死代码检测

**工具映射**:

| 语言       | 工具     | 检测类型           |
| ---------- | -------- | ------------------ |
| Python     | vulture  | 未使用函数/类/变量 |
| TypeScript | ts-prune | 未使用导出         |
| Go         | unused   | 未使用符号         |

**复杂度**:

| 操作           | 时间    | 空间 |
| -------------- | ------- | ---- |
| Python检测     | O(F×n) | O(s) |
| TypeScript检测 | O(F×n) | O(s) |
| Go检测         | O(F×n) | O(s) |

---

### 6. AST相似度算法

**功能**: 基于AST特征向量计算函数相似度

**特征向量**:

- 节点类型分布（标准化）
- 节点深度统计（均值、标准差、最大值）
- 子树结构哈希（Jaccard相似度）
- 标记序列（LCS近似）

**相似度计算**:

```
similarity = 0.4 × cosine_sim + 0.4 × jaccard_sim + 0.2 × token_sim
```

**阈值**: 0.85（超过则触发复用提示）

**复杂度**:

| 操作       | 时间      | 空间 |
| ---------- | --------- | ---- |
| 特征提取   | O(n)      | O(n) |
| 相似度计算 | O(d)      | O(1) |
| 全项目检测 | O(k²×d) | O(k) |

**优化**: 使用LSH可将全项目检测优化至O(k log k)

## 使用示例

### 初始化系统

```python
from core_algorithms import CognitiveScaffoldingSystem

system = CognitiveScaffoldingSystem("/path/to/project")
system.initialize()
```

### 分析修改影响

```python
modified_files = ["src/utils/helper.ts", "src/api/user.ts"]
impact = system.analyze_modification(modified_files)
print(f"Total affected: {impact['impact_report']['summary']['total_affected']}")
```

### 检查代码复用

```python
suggestions = system.check_code_reuse("src/components/Button.tsx:handleClick")
for suggestion in suggestions:
    if suggestion['should_reuse']:
        print(suggestion['recommendation'])
```

### 检测死代码

```python
dead_code = system.detect_dead_code()
for language, report in dead_code.items():
    print(f"{language.value}: {len(report.items)} dead code items")
```

## 算法复杂度汇总

| 算法       | 时间复杂度  | 空间复杂度 | 关键优化     |
| ---------- | ----------- | ---------- | ------------ |
| 符号索引   | O(F×n)     | O(S)       | 多索引哈希表 |
| 依赖图构建 | O(F×n+E)   | O(V+E)     | 邻接表存储   |
| PageRank   | O(k×V²)   | O(V²)     | 稀疏矩阵优化 |
| 影响传播   | O(k×(V+E)) | O(k×V)    | BFS剪枝      |
| 死代码检测 | O(F×n)     | O(s)       | 多工具并行   |
| AST相似度  | O(k²×d)   | O(k)       | LSH加速      |

**符号说明**:

- F: 文件数量
- n: 平均文件行数
- V: 节点数量
- E: 边数量
- S: 符号数量
- k: 修改节点数/函数数
- d: 向量维度
- s: 死代码项数

## 部署建议

1. **增量更新**: 仅重新分析修改的文件
2. **缓存机制**: 缓存AST解析结果和特征向量
3. **并行处理**: 使用多线程并行处理多个文件
4. **内存优化**: 使用生成器处理大型代码库
